== Risks and Technical Debts

[options="header",cols="1,3"]
|===
|Risk / Technical Debt |Description

|Changing requirements
|University processes and requirements may change during development, requiring adjustments in the system.

|Integration complexity
|Connecting the application with existing university systems could cause unexpected technical issues.

|Limited testing time
|Because of the limited project duration, not all scenarios and edge cases may be fully tested.
