== Risks and Technical Debts

[options="header",cols="1,3"]
|===
|Risk / Technical Debt |Description

|Weak module boundaries
|If the modular monolith is not clearly separated into modules, dependencies may grow over time and make the system harder to maintain.

|Integration dependency risk
|The system depends on existing student information and financial systems. If these systems change or provide unstable interfaces, integration problems may occur.

|Security and access control debt
|If roles and permissions are not defined carefully from the start, sensitive student, grade, and financial data could be exposed incorrectly.
|===
